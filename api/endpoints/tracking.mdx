## Overview

The tracking endpoints handle package verification, tracking details retrieval, and package image management for existing packages.

**Base URL**: `https://beehive.gbilogistics.net/api/mobile/v1.0`

**Authentication**: All endpoints require a valid JWT token

## Endpoints

### POST /Tracking/verify-tracking-number

Verify if a tracking number exists in the system and check for duplicate scans.

**Endpoint**: `/api/mobile/v1.0/Tracking/verify-tracking-number`

**Request Body**:

<ParamField path="trackingNumber" type="string" required>
  The tracking number to verify
</ParamField>

<ParamField path="userVerificationDate" type="string" required>
  ISO 8601 timestamp of verification
</ParamField>

**Example Request**:

```json
{
  "trackingNumber": "1Z999AA10123456784",
  "userVerificationDate": "2024-03-06T14:30:00.000Z"
}
```

**Response**:

<ResponseField name="content" type="object">
  Verification result
  
  <Expandable title="content properties">
    <ResponseField name="alreadyExists" type="boolean">
      True if the tracking number exists in the system
    </ResponseField>
    
    <ResponseField name="lastScannedDate" type="string">
      ISO 8601 timestamp of the last scan (if exists)
    </ResponseField>
    
    <ResponseField name="packageId" type="string">
      Internal package ID (if exists)
    </ResponseField>
  </Expandable>
</ResponseField>

**Example Response**:

```json
{
  "code": 1,
  "message": "Success",
  "messageDetail": "Tracking number verified",
  "content": {
    "alreadyExists": true,
    "lastScannedDate": "2024-03-05T10:15:00.000Z",
    "packageId": "pkg_abc123xyz"
  }
}
```

**Usage Example**:

```dart measurement_service.dart:66-90
Future<ApiResponse<VerifyTrackingNumberResponse>> verifyTrackingNumber(
  VerifyTrackingNumberRequest request,
) async {
  try {
    final response = await _http.post<VerifyTrackingNumberResponse>(
      ApiEndpoints.verifyTrackingNumber,
      request.toJson(),
      (json) => VerifyTrackingNumberResponse.fromJson(json),
    );

    if (!response.isSuccessful) {
      return ApiResponse.error(
        messageDetail: response.messageDetail,
        content: const VerifyTrackingNumberResponse.empty(),
      );
    }

    return response;
  } catch (_) {
    return ApiResponse.error(
      messageDetail: null,
      content: const VerifyTrackingNumberResponse.empty(),
    );
  }
}
```

<Note>
  This endpoint is typically called before registering a new package to prevent duplicate entries and warn users about recent scans.
</Note>

### POST /Tracking/register-package-images

Add new images to an existing package.

**Endpoint**: `/api/mobile/v1.0/Tracking/register-package-images`

**Request Body**:

<ParamField path="packageId" type="string" required>
  Internal package ID
</ParamField>

<ParamField path="images" type="array" required>
  Array of image files to add
  
  <Expandable title="image object properties">
    <ParamField path="order" type="int" required>
      Display order (0-based)
    </ParamField>
    
    <ParamField path="encodeContent" type="string" required>
      Base64-encoded image data
    </ParamField>
    
    <ParamField path="fileExtension" type="string" required>
      Image file extension (e.g., "jpg")
    </ParamField>
    
    <ParamField path="fileName" type="string" required>
      Original filename
    </ParamField>
  </Expandable>
</ParamField>

**Example Request**:

```json
{
  "packageId": "pkg_abc123xyz",
  "images": [
    {
      "order": 0,
      "encodeContent": "iVBORw0KGgoAAAANSUhEUgAA...",
      "fileExtension": "jpg",
      "fileName": "photo_1.jpg"
    },
    {
      "order": 1,
      "encodeContent": "iVBORw0KGgoAAAANSUhEUgBB...",
      "fileExtension": "jpg",
      "fileName": "photo_2.jpg"
    }
  ]
}
```

**Response**:

<ResponseField name="content" type="object">
  <Expandable title="content properties">
    <ResponseField name="userMessage" type="string">
      Success message to display to the user
    </ResponseField>
  </Expandable>
</ResponseField>

<Note>
  Images are reordered automatically by the mobile app using drag-and-drop. The `order` field ensures images display in the correct sequence.
</Note>

Delete specific images from a package.

**Endpoint**: `/api/mobile/v1.0/Tracking/delete-tracking-images`

**Response**:

<ResponseField name="content" type="object">

<Warning>
  Image deletion is permanent. The mobile app should confirm the action with the user before calling this endpoint.
</Warning>

## Common Response Structure

All endpoints return a standardized response format:

```json
{
  "code": 1,
  "message": "Success",
  "messageDetail": "Optional detailed message",
  "content": { /* endpoint-specific data */ }
}
```

**Response Codes**:
- `1` = Success
- `2+` = Error (see [Error Handling](/development/error-handling) for full list)
