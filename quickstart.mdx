Get started with LogiScan in just a few steps. This guide will help you set up and run the application quickly.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Flutter SDK** (3.9.2 or higher)
- **Git**
- **Android Studio** or **Xcode** (depending on your target platform)
- **Firebase CLI** (for Firebase configuration)

<Note>
If you haven't installed Flutter yet, check out our [Installation](/installation) guide for detailed setup instructions.
</Note>

## Quick Setup

<Steps>
  <Step title="Clone the Repository">
    Clone the LogiScan repository to your local machine:

    ```bash
    git clone <repository-url>
    cd logiscan
    ```
  </Step>

  <Step title="Install Dependencies">
    Install all required Flutter packages:

    ```bash
    flutter pub get
    ```

    This will install all dependencies defined in `pubspec.yaml`, including:
    - Firebase Core & Crashlytics
    - Provider for state management
    - Mobile scanner for barcode scanning
    - Image picker for camera functionality
    - HTTP client for API communication
  </Step>

  <Step title="Configure Firebase">
    LogiScan uses Firebase for crash reporting and analytics. The Firebase configuration is already included in the project:

    - Android: `android/app/google-services.json`
    - iOS: `ios/Runner/GoogleService-Info.plist` (if applicable)
    - Generated options: `lib/firebase_options.dart`

    <Warning>
    For production deployment, you'll need to replace the Firebase configuration files with your own project credentials.
    </Warning>

    The app will automatically initialize Firebase on startup:

    ```dart lib/main.dart:20-26
    Future<void> main() async {
      WidgetsFlutterBinding.ensureInitialized();
      
      // Initialize Firebase
      await Firebase.initializeApp(
        options: DefaultFirebaseOptions.currentPlatform,
      );
    ```
  </Step>

  <Step title="Configure API Environment">
    Update the API configuration based on your environment. Open `lib/core/config/api_config.dart` and set the appropriate mode:

    ```dart lib/core/config/api_config.dart:15-19
    /// Modo de desarrollo (cambiar a false para producción)
    static const bool isDevelopment = false;

    /// URL base de la API (selecciona según el modo)
    static String get baseUrl => isDevelopment ? devBaseUrl : prodBaseUrl;
    ```

    **Available environments:**
    - Development: `http://100.104.120.121:82`
    - Production: `https://beehive.gbilogistics.net`

    <Note>
    Set `isDevelopment = true` for local testing, or `false` to connect to production servers.
    </Note>
  </Step>

  <Step title="Run the Application">
    Connect your device or start an emulator, then run:

    ```bash
    flutter run
    ```

    For a specific platform:

    <CodeGroup>
    ```bash Android
    flutter run -d android
    ```

    ```bash iOS
    flutter run -d ios
    ```

    ```bash Release Mode
    flutter run --release
    ```
    </CodeGroup>

    The app will launch and display the authentication screen.
  </Step>
</Steps>

## Verify Your Setup

Once the app is running, you should see:

1. **Authentication Screen** - The login page where users can enter credentials
2. **Firebase Connection** - Check your Firebase console for crash reporting initialization
3. **API Connectivity** - The app will attempt to connect to the configured API endpoint

<Note>
The application requires valid credentials to log in. Contact your system administrator for test credentials.
</Note>

## Troubleshooting

### Build Errors

If you encounter build errors, try:

```bash
flutter clean
flutter pub get
flutter run
```

### Firebase Issues

If Firebase initialization fails:

1. Verify `google-services.json` is in `android/app/`
2. Check that Firebase project ID matches in all config files
3. Ensure FlutterFire CLI configuration is up to date

### API Connection Problems

If the app cannot connect to the API:

1. Verify the `isDevelopment` flag in `api_config.dart`
2. Check network connectivity
3. Ensure the API endpoint is accessible from your device/emulator
4. Review logs for specific error messages

<Warning>
When running on an Android emulator, `localhost` refers to the emulator itself, not your host machine. Use `10.0.2.2` instead of `localhost` or configure the appropriate IP address.
</Warning>
