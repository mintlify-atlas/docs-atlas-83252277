## Overview

LogiScan's image processing system handles photo capture, compression, encoding, and storage for package documentation. The system is optimized for mobile devices and network efficiency, with configurable quality settings and size limits.

## Image Capture Configuration

### Quality Settings

Images are captured with the following parameters:

```dart
final XFile? image = await _picker.pickImage(
  source: ImageSource.camera,
  maxWidth: 1920,      // Full HD width
  maxHeight: 1080,     // Full HD height
  imageQuality: 85,    // 85% quality - balance between quality and size
);
```

**Location**: `lib/features/auth/presentation/pages/scan_details_page.dart:82`

### Resolution Rationale

- **1920x1080**: Provides Full HD quality sufficient for label reading and AI processing
- **85% Quality**: Reduces file size by ~50% with minimal visual degradation
- **Result**: Typical image size 200-500KB (down from 1-2MB)

<Note>
  These settings are optimized for warehouse environments where labels and barcodes need to be readable but file size must be minimized for network transmission.
</Note>

## Image Processing Pipeline

### 1. Capture

Images are captured using the `image_picker` package:

```dart
Future<void> _addImageFromCamera() async {
  // Check image limit
  if (_images.length >= _maxImagesPerPackage) {
    ScaffoldMessenger.of(context).showSnackBar(
      SnackBar(
        content: Text(
          'Maximum $_maxImagesPerPackage images allowed. Remove one first.',
        ),
      ),
    );
    return;
  }

  final XFile? image = await _picker.pickImage(
    source: ImageSource.camera,
    maxWidth: 1920,
    maxHeight: 1080,
    imageQuality: 85,
  );
  
  if (image == null) return;

  setState(() => _images.add(image));
}
```

## API Integration

### Upload Package Images

**Endpoint**: `POST /api/mobile/v1.0/ProcessPackage/register-package`

**Request**:
```json
{
  "trackingNumber": "ABC123456789",
  "weight": 15.5,
  "width": 12,
  "length": 10,
  "height": 8,
  "files": [
    {
      "encodeContent": "base64EncodedImageData...",
      "fileExtension": "jpg",
      "fileName": "package_photo_1.jpg"
    },
    {
      "encodeContent": "base64EncodedImageData...",
      "fileExtension": "jpg",
      "fileName": "package_photo_2.jpg"
    }
  ]
}
```

### Update Package Images

**Endpoint**: `POST /api/mobile/v1.0/Tracking/register-package-images`

**Request**:
```json
{
  "packageId": 12345,
  "files": [
    {
      "storedFileId": 67890
    },
    {
      "encodeContent": "base64EncodedImageData...",
      "fileExtension": "jpg",
      "fileName": "new_photo.jpg"
    }
  ]
}
```

<Note>
  When updating images, include existing images by `storedFileId` to keep them, and new images with `encodeContent` to add them.
</Note>

### Delete Package Images

**Endpoint**: `DELETE /api/mobile/v1.0/Tracking/delete-tracking-images`

**Request**:
```json
{
  "storedFileIds": [67890, 67891]
}
```

## Performance Optimization

### Memory Management

<AccordionGroup>
  <Accordion title="Image Compression">
    - Compress at capture time (85% quality)
    - Limit resolution to 1920x1080
    - Use efficient JPEG encoding
    - Result: ~80% file size reduction
  </Accordion>
  
  <Accordion title="Lazy Loading">
    - Load thumbnails on demand
    - Use placeholder images during load
    - Cache decoded images in memory
    - Dispose images when not visible
  </Accordion>
  
  <Accordion title="Network Efficiency">
    - Batch multiple images in single request
    - Use Base64 encoding for API compatibility
    - Implement retry logic for failed uploads
    - Show upload progress indicators
  </Accordion>
</AccordionGroup>

### File Size Comparison

| Setting | Average Size | Upload Time (4G) | Quality |
|---------|--------------|------------------|----------|
| Original (4032x3024) | 2.5 MB | ~8 seconds | Excellent |
| Compressed (1920x1080, 100%) | 800 KB | ~2.5 seconds | Excellent |
| **Optimized (1920x1080, 85%)** | **350 KB** | **~1 second** | **Very Good** |
| Over-compressed (1280x720, 70%) | 150 KB | ~0.5 seconds | Fair |

<Tip>
  The optimized settings (1920x1080 @ 85%) provide the best balance between quality and performance for warehouse operations.
</Tip>

## Common Issues

### Storage Permissions

```xml
<!-- Android: AndroidManifest.xml -->
<uses-permission android:name="android.permission.CAMERA" />
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
```

```xml
<!-- iOS: Info.plist -->
<key>NSCameraUsageDescription</key>
<string>LogiScan needs camera access to capture package photos</string>
<key>NSPhotoLibraryUsageDescription</key>
<string>LogiScan needs photo library access to save images</string>
```

### Upload Failures

- Check network connectivity
- Verify file size limits
- Retry with exponential backoff
- Cache images locally if upload fails

### Memory Issues

- Dispose image controllers after use
- Clear image cache periodically
- Limit simultaneous image processing
- Use image compression libraries

<Warning>
  Always dispose of image resources properly to prevent memory leaks, especially when handling multiple images.
</Warning>
